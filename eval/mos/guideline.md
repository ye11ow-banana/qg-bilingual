# MOS оцінювання питань

Ця інструкція використовується для збірки людських оцінок якості згенерованих питань (Mean Opinion Score, MOS). Оцінюємо **лише питання**, контекст читаємо тільки для перевірки доречності.

## Шкала 1–5
- **5 — Відмінне.** Грамотне, природне формулювання; однозначне; точно відповідає факту в контексті; коректний тип wh.
- **4 — Добре.** Невеликі стилістичні похибки, але питання зрозуміле й релевантне; wh-тип правильний.
- **3 — Задовільне.** Помітні помилки або нечіткість, але зміст можна зрозуміти; wh-тип може бути сумнівним.
- **2 — Погане.** Багато граматичних/стилістичних проблем, сильна неоднозначність або часткова невідповідність контексту.
- **1 — Непридатне.** Фактично неправильне або нерелевантне питання; wh-тип невірний; токсичне чи беззмістовне.

## Критерії
- **Граматика/стиль:** без грубих помилок, природний тон.
- **Чіткість:** однозначне формулювання, без деіксиса ("вони", "там" тощо), без зайвих деталей.
- **Відповідність контексту:** питає про факт, який явно присутній у наведеному контексті.
- **Wh-тип:** відповідає факту (who/when/where/what/why/how або їх UA відповідники).

## Політика «context-only»
Не вигадуємо фактів, не додаємо оцінних ярликів. Якщо інформації в контексті немає або питання не відповідає факту — ставимо низьку оцінку.

## Приватність/етика
- Не анотуємо приклади з персональними даними чи чутливими атрибутами; якщо такі трапляються, вони мають бути замінені на маркери `[PERSON]`, `[LOCATION]` тощо.
- Анотатори бачать лише контекст і питання, без авторів або інших метаданих.

## Флаги (можна кілька)
- `grammar` — граматичні/стилістичні помилки.
- `ambiguous` — неоднозначне або занадто загальне.
- `ooc` — не відповідає контексту (out-of-context).
- `wrong_wh` — неправильний wh-тип.
- `too_long` / `too_short` — надто довге/коротке.
- `toxic` — образливе/упереджене.

## Формат відповіді
Див. `schema.json`. Зберігайте один JSONL-файл на анотатора: `results/mos_raw_annotator_<id>.jsonl`.
